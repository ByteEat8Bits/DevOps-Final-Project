應用層度量指標：

1. 請求總數（TotalRequests）
    o 可以用來監控資料收集腳本的運行頻率，並確保自動化資料收集機制正常工作。
2. 錯誤總數（TotalErrors）
    o 用於識別資料抓取、分析過程中的錯誤（例如 API 請求失敗或資料處理錯誤），確保數
      據展示的準確性。
3. 請求延遲（RequestLatency）
    o 對於定時請求的專案來說，可以觀察請求的延遲情況，特別是從 API 或數據源抓取數據
      的延遲，確保及時性。
4. 任務失敗率（TaskFailureRate）
    o 專案中某些自動化流程（如資料抓取、數據分析）的失敗率，可以幫助我們及早發現並
      解決潛在問題

網絡層度量指標：

1. 網絡延遲（NetworkLatency）
    o 監控 API 請求的延遲時間的重要指標，特別是資料抓取過程。如果網絡延遲較高，可能
      影響數據的即時性和準確性。
2. HTTP狀態碼數量（HTTPStatusCodeCount）
    o 用於追蹤每次API請求的結果，例如200表示成功，404表示資源未找到，500表示伺服
      器錯誤等。這有助於識別網絡或服務器問題，確保數據請求的穩定性。
3. 網絡入站流量（InboundTraffic）和網絡出站流量（OutboundTraffic）
    o 監控網絡的入站與出站流量可以幫助了解數據傳輸的頻率和數據量，特別是當請求數據
      量大、頻率高時。這樣有助於優化請求策略，避免不必要的網絡消耗。

使用者互動度量指標：

1. 使用者操作量（UserActions）
    o 追蹤用戶的操作次數（例如搜尋次數、切換股票、點擊圖表等），可以幫助了解哪些功
      能最受歡迎，哪些功能的使用頻率較低，便於進一步優化。
2. 會話長度（SessionDuration）
    o 測量每個用戶在頁面停留的時間可以幫助分析用戶的互動深度。如果會話長度較長，說
      明用戶在數據展示頁面上有較多的互動和參與；會話長度較短可能意味著用戶快速退出
      或未找到所需信息。
      
其它：因為本次的專案並沒有設計登入系統，所以與其有關的指標就沒有考慮使用。
